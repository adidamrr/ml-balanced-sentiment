# Balanced sentiment experiment

Это небольшой эксперимент для проверки перекоса в агрегации тональности отзывов. Baseline считает тональность по словам во всём пуле текстов, lexicon сначала классифицирует каждый отзыв отдельно, а ensemble добавляет Groq LLM только для сложных случаев.

## Что нужно на машине

Python 3.10+.

```bash
pip install -r requirements.txt
```

Основная демонстрация не требует LLM, GPU, внешних API и скачивания моделей. LLM используется только в опциональном ensemble-подходе через Groq.

## Как воспроизвести

Запустить тесты:

```bash
pytest
```

Тесты лежат в `tests/`: `test_lexicon_sentiment.py` и `test_ensemble_sentiment.py`. В каждом по 10 одинаковых сценариев: пустые входы, мусор, короткие positive/negative/neutral, mixed, русский и английский, отрицания, оценки/emoji и баланс `45/45/10` с длинными negative-отзывами. Для ensemble сарказм проверяется через fake Groq client без реального API-вызова.

Запустить тихий JSON-отчёт:

```bash
python tests/run_tests_json.py
```

Результат будет записан в `test_results.json` без подробного вывода в терминал.

Запустить сравнение трёх подходов без ноутбука:

```bash
python analysis/compare_approaches.py
```

Если задан `GROQ_API_KEY`, ensemble будет использовать Groq LLM. Если ключ не задан, ensemble работает в fallback-режиме без API-вызовов.

Открыть и выполнить ноутбук:

```bash
jupyter notebook analysis/balanced_sentiment.ipynb
```

Ноутбук подходит для Google Colab. Если загрузить `.xlsx` во временное хранилище Colab, он найдёт файл в `/content` и покажет краткую сводку по комментариям. Основные выводы не зависят от этого файла: демонстрация до/после строится на синтетических сценариях с известными метками.

Для Colab достаточно загрузить только `analysis/balanced_sentiment.ipynb`. Папки `src`, `tests` и `analysis` дополнительно копировать не нужно: весь код, графики и JSON-тесты встроены прямо в notebook.

## Файлы подходов

`src/baseline_sentiment.py` содержит старый baseline: подсчёт sentiment-сигналов по словам. Это специально оставлено как проблемный подход для сравнения.

`src/lexicon_sentiment.py` содержит review-level лексикон: очистка текста, обработка оценок и emoji, русский и английский словарь, отрицания, классификация одного отзыва и распределение по отзывам.

`src/ensemble_sentiment.py` содержит подход `lexicon first, LLM for uncertain reviews`: простые отзывы решает лексикон, сложные проверяет Groq `llama-3.1-8b-instant`.

`analysis/compare_approaches.py` строит сценарии A/B/C, sensitivity-таблицу и печатает сравнение `baseline / lexicon / ensemble`.

## Логика

Baseline имитирует старую проблему: все отзывы склеиваются в общий пул, затем считаются sentiment-слова. Длинный негативный отзыв даёт больше негативных слов, чем короткий позитивный отзыв даёт позитивных, поэтому итог может уехать в сильный негатив даже при одинаковом количестве positive и negative отзывов.

Lexicon решает задачу на уровне одного отзыва. Для каждого текста считается простой лексиконный score с обработкой отрицаний вроде `не понравилось`, `не плохо`, `not bad`. Затем отзыв получает один label: `positive`, `negative` или `neutral`. Итоговое распределение считается по количеству отзывов, поэтому длина текста сама по себе не увеличивает вес отзыва.

Neutral не считается остатком до 100%. Отзыв попадает в `neutral`, если sentiment-score близок к нулю, если в тексте нет понятных sentiment-сигналов, или если положительные и отрицательные сигналы взаимно компенсируются.

Лексикон расширен базовыми словами на русском и английском. Это не полноценная multilingual-модель, а простая воспроизводимая эвристика для демонстрации подхода.

Короткие оценки вроде `5/5`, `10/10`, `1/5` и понятные смайлики тоже обрабатываются как отзывы. Пустые строки, точки, одиночные буквы и неизвестные emoji игнорируются.

## Ensemble

Отдельный файл `ensemble_sentiment.py` добавляет ещё один подход: `lexicon first, LLM for uncertain reviews`. Простые уверенные отзывы решает лексикон, а Groq `llama-3.1-8b-instant` вызывается только для пограничных случаев: mixed, длинных, англоязычных, саркастичных или отзывов с одновременно positive и negative сигналами.

В Colab можно задать ключ так:

```python
import os
os.environ["GROQ_API_KEY"] = "your_key"
```

LLM в ensemble получает один отзыв за один запрос и возвращает JSON вида `{"label": "neutral", "confidence": 0.8, "flags": ["mixed"]}`. Настройки: `temperature=0`, `top_p=1`, `max_tokens=60`, `response_format={"type": "json_object"}`.

Если `GROQ_API_KEY` не задан, ensemble не ломается и использует fallback на лексикон. В результате дополнительно показываются `llm_used` и `fallback_count`, чтобы было видно, сколько отзывов реально потребовали LLM.

В тестах используется fake Groq client только для проверки логики без сети и без списания API-лимитов. В рабочем запуске используется настоящий Groq client.

## Ограничения

Качество лексикона зависит от количества ключевых слов в словаре. Ирония и сарказм криво обрабатываются. При перечислении плюсов и минусов трудно прийти к одному выводу: позитивный, нейтральный или негативный этот отзыв. Требуется внешний API для использования модели
